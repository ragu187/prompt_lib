# Task Planning and Organization

## Category
Productivity

## Description
Break down complex tasks into manageable steps, create action plans, and organize work effectively.

## Prompt

```
Help me plan and organize the following task:

**Task/Project:**
[TASK_DESCRIPTION]

**Goal:**
[DESIRED_OUTCOME]

**Deadline:**
[TIMELINE]

**Available Time:**
[TIME_AVAILABLE]

**Resources:**
[RESOURCES_AND_CONSTRAINTS]

**Current Status:**
[WHAT_IS_DONE]

Please provide:
1. **Task Breakdown**: Break the task into smaller, manageable subtasks
2. **Prioritization**: Order tasks by importance and dependencies
3. **Time Estimates**: Realistic time allocation for each subtask
4. **Schedule**: Suggested timeline/schedule
5. **Milestones**: Key checkpoints to track progress
6. **Risk Factors**: Potential obstacles and mitigation strategies
7. **Success Criteria**: How to know when each step is complete
```

## Variables

- `[TASK_DESCRIPTION]`: The main task or project to plan
- `[DESIRED_OUTCOME]`: What success looks like
- `[TIMELINE]`: Deadline or target completion date
- `[TIME_AVAILABLE]`: How much time you can dedicate
- `[RESOURCES_AND_CONSTRAINTS]`: Available resources and limitations
- `[WHAT_IS_DONE]`: Current progress or starting point

## Example Usage

### Input
```
Help me plan and organize the following task:

**Task/Project:**
Launch a personal blog with 10 articles

**Goal:**
Have a fully functional blog with quality content ready to attract readers

**Deadline:**
6 weeks from now

**Available Time:**
10 hours per week

**Resources:**
- Basic web development skills
- Writing experience
- $100 budget

**Current Status:**
Haven't started yet, but have some topic ideas

Please provide:
1. **Task Breakdown**: Break the task into smaller, manageable subtasks
2. **Prioritization**: Order tasks by importance and dependencies
3. **Time Estimates**: Realistic time allocation for each subtask
4. **Schedule**: Suggested timeline/schedule
5. **Milestones**: Key checkpoints to track progress
6. **Risk Factors**: Potential obstacles and mitigation strategies
7. **Success Criteria**: How to know when each step is complete
```

### Expected Output
A detailed project plan with weekly milestones, task breakdown (setup, design, content creation, etc.), time allocation, and contingency plans.

## Tips

- Be realistic about available time and resources
- Include buffer time for unexpected issues
- Identify dependencies between tasks
- Set clear, measurable milestones
- Consider what could go wrong and plan accordingly
- Can be used for projects, goals, events, or complex tasks
